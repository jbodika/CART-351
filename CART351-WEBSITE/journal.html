WEEK 1

This first class was mostly about going over the course outline and expectations. We didn't dive into new material yet, but it helped to see what the semester would look like and what kinds of projects we'd be working on.

WEEK 2

This week we had an introduction to Python. We reviewed creating variables, doing basic math, formatting output, taking user input, writing functions (including callback functions), and using if/else statements. It covered a lot, but it felt manageable.

I also learned about MiniConda, which is a tool for managing Python dependencies and creating isolated virtual environments. It seems really useful in theory, although I'm still not totally confident that I'm using it correctly yet. Since I already have some experience with Python, this class mostly felt like a refresher, which was nice.

WEEK 3

This week we learned how to integrate web APIs into a Python project in order to manipulate and analyze data. I'd never used a weather API with Python before, so it was interesting to see how external data can be pulled in and worked with.

WEEK 4

We were introduced to Flask, a Python framework used to create web applications. We ran our first web server and learned how to create dynamic pages by passing input between routes. Seeing Python work directly with a browser was really exciting and made things less static.

WEEK 5

CANCELLED CLASS DUE TO STRIKE.

WEEK 6

We continued exploring Flask and focused more on how it can be used creatively, not just functionally. It was helpful to see how flexible Flask is and how it can support more experimental or interactive ideas.

WEEK 7

This week we kept working with Flask and Python, running web servers and creating dynamic pages with user input. Repeating these steps helped reinforce how routing and data passing actually work.

WEEK 8

We worked with JSON and form data using method arguments. I was already familiar with using JSON in projects, but it was useful to revisit it in the context of Flask. We also learned how to manipulate and store user data in JSON files, which feels very practical for smaller applications.

WEEK 9

This week we had a lecture on MongoDB and how to integrate it into Flask applications. MongoDB is a schema-flexible database, and its interface feels pretty intuitive. We went through some examples, which really helped make sense of it. I liked that MongoDB allows you to create, read, update, and delete data in real time.

WEEK 10

Today's class was more interactive. We walked around the classroom reviewing each other's Flask projects. I really enjoy this format because we get feedback, talk through ideas, and see how everyone approaches the same assignment differently. It's also just fun to explore other people's projects. We also reviewed sockets for real-time user connections, which seems super useful for interactive applications.

WEEK 11

We continued working with MongoDB and focused on visualizing data on a web page. We were shown how MongoDB can be integrated with p5.js to create interesting visualizations. This made the data feel more expressive and creative rather than just functional.

WEEK 12

This week we were assigned an exercise that took way longer than I expected. Coming up with three different visualizations that all showcased the same data was harder than I thought. Even though it was challenging, I think I managed to create something I actually like, which made the effort feel worth it.
